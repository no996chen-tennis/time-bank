# 时间银行 — 设计规范

> Version: 1.0 | Date: 2026-04-19 | 配套 PRD V1.2
>
> 本文是 UI/UX 设计师的交付标准。任何一个设计决策如果违反下面原则，必须在 PR/Figma 里解释"为什么这次要破例"。

---

## 一、设计哲学

### 三条原则（按优先级，冲突时照此取舍）

1. **温暖 > 精致**
   产品是在触碰亲情、时间、生死这些重量话题，冷硬、极客、赛博风会让人关上心门。宁可视觉"软"一点，也不要"酷"。

2. **呼吸感 > 信息密度**
   每一屏只让用户做一件事。大量留白、行距放松、数字不挤。用户应该能在这个 App 里慢下来。

3. **数字有温度 > 数字准确**
   "你还能见约 92 次"比"你还能见 91.7 次"重要。估算感比精确感更符合产品气质。永远带"约"字，永远不要 99 位小数。

### 一条反面禁令

**不要任何 KPI / 任务打卡 / 进度条的视觉语言**。

时间银行不是习惯养成 App。不要出现：连续签到、本周目标、完成率、徽章、排行榜、成就系统。看到这些视觉 → 立刻删除或重设计。

## 二、色板

### 基础色

| 角色 | Hex | 用途 |
|------|-----|------|
| `bg` | `#0A0A0F` | 整体背景（深色，默认态）|
| `bg-light` | `#F5F2EC` | 浅色模式背景（留白米色）|
| `card` | `#16161F` | 卡片背景（深色）|
| `card-light` | `#FFFFFF` | 卡片背景（浅色）|
| `card-2` | `#1C1C26` | 次级卡片 / 输入框 |
| `border` | `#22222E` | 分隔线、边框 |
| `border-soft` | `#2A2A38` | 柔和分隔 |
| `text` | `#F0EDE6` | 主文字（暖米白，不是纯白）|
| `text-2` | `#8A8797` | 次级文字 |
| `text-3` | `#605E6E` | 更次文字、占位符 |

### 维度色（仅作为 icon / accent，不作为大面积色块）

每个维度一个暖色系，**不饱和**，偏柔和：

| 维度 | Hex | 使用场景 |
|------|-----|---------|
| 陪父母 `rose` | `#D4847C` | 胸前带着体温的暖色 |
| 陪孩子 `warm` | `#E8C47C` | 朝阳的暖金色 |
| 陪伴侣 `lavender` | `#A088C4` | 一起看过的黄昏紫 |
| 运动 `sage` | `#7CB89E` | 晨跑路过的灌木绿 |
| 创造 `sky` | `#7CACD4` | 清晨窗外的天蓝 |
| 自由 `peach` | `#D4A07C` | 温暖的惰散 |

### 语义色

| 角色 | Hex | 场景 |
|------|-----|------|
| `success` | `#7CB89E`（复用 sage） | 存入成功 |
| `warn` | `#E8A77C` | 存储空间告警 |
| `danger` | `#D4847C`（复用 rose） | 删除按钮 |

**核心规则：** 色彩**永远不单独传达信息**。消耗层数字和存储层数字不能只靠颜色区分——必须同时有标签（"还能存入" / "已存入"）、图标（↓ / ↑），避免色盲用户出问题。

## 三、字体

### 字体族

```
iOS System:
  中文：PingFang SC
  英文/数字：SF Pro Text / SF Pro Display / SF Pro Rounded（大数字用 Rounded）

Web 原型：
  'Inter' + 'Noto Sans SC', -apple-system, sans-serif
```

特殊场景可考虑 **Serif（思源宋体 / Source Han Serif）** 用于：
- 空态插画中的诗意文案（"这里还没有被存下的时刻"）
- "一封月度信"回顾文案（模拟手写信感）

### 字阶（iOS Dynamic Type 友好）

| 角色 | 默认 Size | Weight | 行高 | 用例 |
|------|----------|--------|------|------|
| Display | 44pt | 200 (Ultralight) | 1.1 | 维度详情页大数字 |
| Hero | 34pt | 300 (Light) | 1.2 | 主页总账户数字 |
| Title-1 | 22pt | 500 (Medium) | 1.3 | 时刻详情标题 |
| Title-2 | 16pt | 600 (Semibold) | 1.4 | 导航栏标题、模块标题 |
| Body | 14pt | 400 (Regular) | 1.7 | 长文本、笔记 |
| Body-Sm | 13pt | 400 | 1.6 | 次级内容、时刻卡片文字 |
| Caption | 12pt | 400 | 1.5 | 元信息、时间 |
| Micro | 10pt | 500 | 1.5 | 标签、Uppercase Label（需加 letter-spacing 1.5）|

**关键：** 大数字用 200/300 字重（不是 700），细瘦的字反而更高级、更不焦虑。厚重字体会让"剩余时间"变得像警告。

## 四、间距 & 圆角

### Spacing Scale（8pt Grid）

```
xs  = 4pt
sm  = 8pt
md  = 12pt
lg  = 16pt    ← 默认容器内边距
xl  = 20pt
2xl = 28pt
3xl = 40pt
```

### 圆角

```
radius-sm = 8pt    输入框、小图标容器
radius-md = 12pt   按钮
radius-lg = 16pt   卡片（默认）
radius-xl = 22pt   hero 卡片、模态框顶部
radius-2xl = 36pt  全屏模态手机外框
full       = 100pt 胶囊 / chip
```

**卡片的默认圆角统一 14-16pt**，全局一致性比个性更重要。

## 五、阴影 & 层级

深色模式下几乎不用投影，**用边框 1px `--border` 区分层级**。

浅色模式（V1 可以先不做，Light Mode 放 V1.5）：
```
shadow-sm:  0 1px 2px rgba(0,0,0,0.04)
shadow-md:  0 4px 12px rgba(0,0,0,0.06)
shadow-lg:  0 8px 24px rgba(0,0,0,0.08)
```

**禁用：** 多层投影、彩色投影、Neumorphism（拟物浮雕）。这些会让产品显得陈旧。

## 六、动效原则

### Spring 为主，不要 Ease

iOS 17+ 优先用 `Animation.spring(response: 0.5, dampingFraction: 0.8)`，拒绝线性/Ease 曲线。Spring 有"物理感"，符合温暖调性。

### 动效时长

| 动作 | 时长 |
|------|------|
| 微交互（按钮、icon） | 0.15-0.2s |
| 页面推进 | 0.35s（系统默认） |
| 存入成功动效 | 0.8-1.2s（特殊时刻可以"慢"） |
| Widget 刷新 | 不需要动效（系统负责） |

### 存入成功动效（产品的"签名动效"）

1. 点"保存" → 按钮按下缩 0.95
2. 表单内容淡出 300ms
3. 一个光点（金色圆）从屏幕中央升起到右上角"已存入"数字位置
4. 光点消失时，对应维度的已存入数字从 `X` → `X+1` 做 roll-up 动画
5. 右上角数字轻微发光 400ms 后归位

这个动效要精心做，它是产品唯一一次"情绪高潮"。

### Reduce Motion

开启"减弱动态效果"时，存入动效降级为简单淡入，光点不再飞行。

## 七、核心组件规范

### 1. 时间维度卡片（Dimension Card）

**结构：**
```
┌────────────────────────────────────────┐
│  [icon] 陪父母                      ›  │  ← 14pt Medium, icon 28×28
├────────────────────────────────────────┤
│  还能存入       │      已存入          │
│   552h         │      56h           │  ← 20pt Medium，icon 22pt
│  约 92 次见面   │      12 个瞬间      │  ← 10pt, text-3
└────────────────────────────────────────┘

padding:  14pt 16pt
radius:   14pt
border:   1px solid border
gap:      卡片之间 10pt
```

### 2. 时刻卡片（Moment Card，时间线列表项）

```
┌──────────────────────────────────────┐
│ [64×64缩略图] │ 标题 (13pt Medium)    │
│              │ 元信息 (11pt text-2)  │
│              │ 笔记首句 (11pt text-3)│
└──────────────────────────────────────┘
```

- 缩略图右上角小角标显示总媒体数（"9"）
- 视频在缩略图上叠一个小 ▸ 图标

### 3. 按钮

**Primary（主按钮）：**
- 背景：`linear-gradient(135deg, var(--warm) 0%, var(--rose) 100%)`
- 文字：`#1a1a24` 14pt Semibold
- 圆角 14pt，内边距 14pt
- 高度约 48pt（满足 44×44 触控）

**Secondary（次按钮）：**
- 背景 `var(--card)`，border 1px `var(--border)`
- 文字 `var(--text)` 13pt Medium

**Ghost（文字按钮）：**
- 仅文字，颜色 `var(--warm)`，13pt Medium

**Danger（危险按钮）：**
- 文字 `var(--rose)`，背景同 Secondary

### 4. 表单字段

- Label: 12pt `var(--text-2)` uppercase letter-spacing 0.5
- Value: 13pt `var(--text)`
- 行高 40pt（触控）
- 行之间 1px `var(--border-soft)` 分隔线

### 5. Chip / Tag

- padding 5×11pt
- radius 100pt
- font 11pt
- 默认：bg `var(--card)`，border 1px `var(--border)`
- Accent: bg `rgba(232,196,124,0.1)`，border `rgba(232,196,124,0.25)`，text `var(--warm)`

## 八、插画 & 图标

### 原则

- **一次只画一条线**（细线条插画 > 填充色块）
- 多用"暗示式"意象：光晕、云、波纹、剪影
- 避免：卡通、拟人化动物、emoji 充数、过度装饰

### 推荐参考

- Headspace 的静态插画风格（圆润、平面、暖色）
- Calm 的光晕质感
- Tot 的极简图标
- **反面教材**：Duolingo 的鲜艳卡通（方向完全错）

### 图标

- 维度 Icon 使用**单色符号** + 维度色：♡（父母）、◐（孩子）、◎（伴侣）、✦（创造）、⟳（运动）、◇（自由）
- 功能 Icon 使用 **SF Symbols**（iOS 原生，免费、一致）
- 尺寸：14pt、16pt、22pt 三档

## 九、Dark Mode / Light Mode

**V1 优先深色**（深色更符合"夜深人静翻看时间银行"的使用场景）。

**V1.5 加浅色：**
- 背景 `#F5F2EC` 米白（不是纯白，降低刺激）
- 卡片 `#FFFFFF` + 1px border `rgba(0,0,0,0.06)`
- 主文字 `#1A1A24`
- 维度色不变，但 Opacity 降 10%

两种模式下所有语义都一致，不要在 Light 下突然改成另一种色板。

## 十、Widget 设计规范

### 尺寸

| 尺寸 | 场景 | 建议内容 |
|------|------|---------|
| Lock Screen Circular | 锁屏胶囊 | 纯数字 + 单字单位（如 `552` / `h`）|
| Lock Screen Rectangular | 锁屏横条 | 数字 + 一句文案（"你和爸还能共度 552h"）|
| Home Small | 桌面小 | 1 个维度 + 数字 + 轮换文案 |

| Home Large | 桌面大 | 4 个维度 grid + 顶部总账户 |
| StandBy | 充电桌 | 大字数字（同锁屏横条放大）|

- 数字字重 **必须用 SF Pro Rounded**（Widget 场景下圆体数字更温暖）
- 文案行高放宽到 1.5，避免太紧

## 十一、空态 & 错误态视觉

空态不是"暂无数据"，而是一次对话。

**结构：**
```
┌──────────────────────┐
│                      │
│    [小插画 60×60]     │
│                      │
│  温暖的一句提示       │
│  (14pt, text-2)      │
│                      │
│     [CTA 按钮]        │
│                      │
└──────────────────────┘
```

- 插画永远比错误图标温暖
- 文案不说"暂无"、"无数据"、"Nothing here"
- 必有 CTA（告诉用户下一步做什么）

## 十三、Do / Don't 快速参考

| ✅ Do | ❌ Don't |
|------|---------|
| 数字用 Light / Ultralight 字重 | 数字用 Bold / Black 字重 |
| "约 92 次" | "92.3 次"（精确到小数吓人） |
| 温暖色 + 低饱和 | 鲜艳色 + 高饱和 |
| 大留白、行距 1.6+ | 信息密集、间距局促 |
| 单色线条插画 | 表情/emoji 充当插画 |
| 用"瞬间"称呼存储层条目 | 用"打卡"/"次数"称呼 |
| 进度以"小时"为单位累积 | 进度用百分比条表示 |
| 存入成功动效"光点上升" | 庆祝特效 / 五彩碎屑 |
| 空态有对话、有 CTA | "暂无数据"四字了事 |
