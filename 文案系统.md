## 一、Tone of Voice（声音）

### 三个词定义 Tone

**温柔 · 成熟 · 不煽情**

- **温柔**：说的像家里人的语气，不是品牌方的语气
- **成熟**：不说"亲～"、不用波浪号、不加多余 emoji
- **不煽情**：把事实说得温暖，而不是用形容词堆砌情绪

### 永远写的 vs 永远不写的

| ✅ 永远这么写 | ❌ 永远不这么写 |
|-------------|---------------|
| "你还能和爸妈一起度过约 552 小时" | "你只剩 552 小时可以陪爸妈了！！" |
| "这一刻，以后会是值得回想的时光" | "珍惜当下，莫负韶华✨" |
| "存入时间银行" | "保存到我的时间胶囊 💎" |
| "今天有什么值得存下来的吗" | "快来签到打卡领奖励🎁" |
| "你已存入 56 小时" | "恭喜你！累计打卡 56 次 🎉" |
| "周末回家看爸妈了吗？" | "家人永远是你的避风港❤️" |
| "还没有被存入的瞬间" | "暂无数据" |
| "这段时间只属于你自己" | "这是你的 'ME TIME' 哦~" |

### 人称规则

- **第二人称"你"** 是默认（像朋友和你说话）
- **第一人称"我"** 只出现在用户自己填的内容里
- **品牌不用"我们"**（不要让用户意识到有公司存在）
- **时间银行本身不自称** —— "时间银行" 4 个字只是 App 名字，行文里尽量用"这里"、"这本账户"

### 语气校准

读出声，如果读起来像：
- ✅ 一位低声的朋友 → 对
- ✅ 一位温和的长辈 → 对
- ❌ 一位热情的客服 → 错（太热情）
- ❌ 一位文艺博主 → 错（太诗意）
- ❌ 一位增长黑客 → 错（太功利）

## 二、核心文案库

### 2.1 App 核心定位（商店 & 引导用）

**一句话定位（App Store 副标题，30 字以内）：**

主推：
> 把有意义的时间，存进来。

备选：
> 重要的时间，让它留下。
> 一个让时间慢下来的地方。
> 你的时间，值得被感受。

**三句话介绍（商店描述开头）：**

> 时间在流逝，但被感受过的时间会留下。
>
> 时间银行让你看到你和父母、孩子、伴侣还能共度多少小时，也让你把那些真正有意义的片段 —— 带着照片、视频和文字 —— 永久存入你的账户。
>
> 不是打卡，不是焦虑。只是一个容器，安静地记下那些以后会想起的时刻。

### 2.2 引导流程文案（V1.3.2 定稿 · 对齐 M2 实际实现的 4 步）

V1 onboarding 是 4 步线性流程：**Step 1 生日基础 → Step 2 关系 chip → Step 3 条件化细节 → Step 4 完成 + Pre-Permission 合并**。无独立"品牌启动屏"（原 V1.2 设计里的 "时间在流逝 / 被感受过的时间会留下" slogan V1 暂不展示，V1.1+ 候选加回）。

---

**Step 1 — 生日 / 性别 / 预期寿命：**

Header 主标题：
> 先从你开始。

Header 副文案：
> 这些只会存在你的手机里。

字段 label：
- `生日` · DatePicker wheel 风格
- `性别` · 4 选 1（chip 形式）· 选项：`男` / `女` / `其他` / `不想说`
- `预期寿命` · Slider 60-120 step 1，默认 85 · 实时显示 `预期 N 岁`

按钮：
- `下一步`（当 `生日 <= 今天` 时 enabled；未来生日 disabled）

**Step 2 — 关系 chip 多选（0-4 个）：**

Header 副文案：
> 不用太准，估个数就好。

Chip（2×2 grid，每个 chip 可独立 toggle；0 个也允许继续）：
- `父母`（SF Symbol `person.2.fill`）
- `伴侣`（SF Symbol `heart.circle.fill`）
- `孩子`（SF Symbol `figure.2.and.child.holdinghands`）
- `独处也是一种时间`（SF Symbol `moon.stars.fill`）

按钮：`上一步` / `下一步`（永远 enabled）

---

**Step 3 — 条件化细节（按 Step 2 勾选展示子表单）：**

Header 主标题：
> 不用太准，估个数就好。

Header 副文案：
> 随时可以改。

按 `OnboardingRelationship` 勾选动态展示 `DetailCard` 子表单（选中了才显示对应 card）：

**父母子表单**（勾了 `.parents`）· card 标题："父母"：
- 父亲 / 母亲各自的"填写"/"清除" toggle · 选"填写"后展开 `生于 N 年`（Stepper，范围 1920 到今年-18）
- `每年见 N 次`（Stepper 1-52，默认 4）
- `每次见面大约 N 小时`（Slider 0.5-24 step 0.5，默认 6）

**孩子子表单**（勾了 `.children`）· card 标题："孩子"：
- 空态提示：`至少添加一个孩子后，就可以继续。`
- 每个孩子 row：`孩子 N 生于 N 年` + 删除按钮（trash icon · `Color.tbDanger`）
- 底部：`+ 添加孩子` 按钮

**伴侣子表单**（勾了 `.partner`）· card 标题："伴侣"：
- `伴侣出生于 N 年`（Stepper）
- `平均每天共处 N 小时`（Slider 0-16 step 0.5，默认 4）

**独处 acknowledgement**（勾了 `.solo`）· card 标题："独处"：
> 独处也是一种时间。这份时间我们也会帮你记下来。

**空态 card**（一个 chip 都没勾）· card 标题："之后再补"：
> 你可以直接继续。家人信息随时能在设置里补上。

按钮：`上一步` / `下一步`（**仅**勾了 `.children` 但 `children.count == 0` 时 disabled；其他情况 enabled）

---

**Step 4 — 完成 + 通知 Pre-Permission（合并一屏）：**

完成 Header 主标题：
> 做好了。

完成 Header 副文案：
> 你将看到你和重要的人还能共度多少时间。遇到值得记住的瞬间，也可以存进来 —— 它们不会被时间带走。

Pre-Permission 子 card（`Color.tbBg2` + `TBRadius.lg` 圆角，视觉上跟完成文案分开）：

主标题：
> 每天早上轻轻提醒你一次 —— 不催，不推销。

副文案：
> 只是帮你记得：今天也有一些片段，值得被留下。

按钮（纵向堆叠，主次分层）：
- 主按钮：`好，开启提醒` · 点击 → `UNUserNotificationCenter.requestAuthorization` + 保存 UserProfile + 路由到主页
- 次按钮：`以后再说` · 点击 → 跳过权限请求 + 保存 UserProfile + 路由到主页（V1.1 设置页可开启通知）

两按钮都在操作进行时 `disabled` + `opacity 0.5`（防重入）。

**总账户副文案（`你已存入 128 小时` 下方）：**

- 累计 0h（空态）：`还没有被存入的时刻。下次有值得留下的片段，来这里看看。`

**时间账户卡片副文案（Label "还能共度" / "已存入" 下方）· V1.3.2 定稿 · 详见 PRD §22.3.1：**

| 时间账户 | 消耗层副文案（"还能共度" / "未来" 下方） | 存储层副文案（"已存入" 下方） |
|------|---------|---------|
| 陪父母 (parents) | 约 N 次见面 | N 个瞬间 |
| 陪孩子 (kids) | 每周约 N 小时 | N 个瞬间 |
| 陪伴侣 (partner) | 每天约 N 小时共处 | N 个瞬间 |
| 运动 (sport) | 每周约 N 小时 | N 个瞬间 |
| 创造 (create) | 每周约 N 小时 | N 个瞬间 |
| 自由 (free) | 占清醒时间约 N% | N 个瞬间 |

**存储层量词铁律**："已存入"下方**只能**是"N 个瞬间"，不得出现"次锻炼 / 个作品 / 个片段"等变体（违反 PRD §21 存储层严格禁用词表）。

## 三、Widget 文案轮换池

### Widget 内容库（按时间账户分类，每天自动轮换）

**陪父母池（N = 15；所有文案遵守"观察式而非命令式"原则）：**
1. `你和爸妈还能一起度过约 552h`
2. `上次见爸妈是 X 天前`
3. `爸爸今年 63 岁了`
4. `已经存下了 12 个和爸妈的瞬间`
5. `也许这周会有一个适合说"最近好吗"的时刻` ← 原"周末打个电话吧"
6. `如果你愿意，找个安静的时候听听他们的声音` ← 原"他们也在等你打电话"
7. `再多一次回家，都是赚的`
8. `你上次存入的瞬间："杭州西湖 48 小时"`
9. `妈妈最喜欢你拍的哪张照片？`
10. `下次见面，也许可以多看一看他们现在的样子` ← 原"记得多看看他们的脸"
11. `他们的样子在慢慢变，像四季那样` ← 原"白头发，你有多久没注意到了"（去掉 guilt）
12. `再有 X 年，爸爸就 70 了`
13. `一年里能一起见面的那些时刻，慢慢会变得更珍贵` ← 原"每年 4 次的见面，就是你能给的最贵的礼物"
14. `时间银行里，陪父母这一栏存了 56h`
15. `这些被你认真留下的时间，已经很珍贵了` ← 原"你已经做到了，比大多数人都多"

**陪孩子池（N = 15）：**
1. `女儿再长 3 年，就不会扑着让你抱了`
2. `这是她人生中最爱你的 10 年`
3. `你存下了 8 个她的瞬间`
4. `他/她今天说了什么让你笑的话`
5. `过两年你会怀念现在的他/她`
6. `黄金陪伴期还剩 7.2 年`
7. ... (持续扩展)

**陪伴侣池（N = 10）：**
1. `你们还能一起看约 14,600 次日落`
2. `上次两个人吃饭是 X 天前`
3. ...

**今日觉知池（不绑定时间账户，随机池，N = 20）：**
1. `这是你的第 12,483 个早晨`
2. `这杯水是你这辈子的第 N 杯`
3. `现在这一刻，以后会是"过去"`
4. `过去 24 小时有什么值得存下来的？`
5. `今天也过去了一些时间。你想留下哪一小段？` ← 原"你今天比昨天多活了一天"
6. `今天的一些时间，会变成以后的回想` ← 原"你今天比昨天离那些人又远了一天..."（存在主义敲打过重）
7. `一杯咖啡的时间，也是时间`
8. `窗外的光线，以后你会想念的`
9. `试试记得今天早晨的味道`
10. `被感受过的时间，永远属于你`
11. `今天的光，刚好落在今天` ← 原"今天的光，以后不会再有了"（guilt 色彩减半档）
12. `不必记得所有，留下一两件就好`
13. `最近的哪一天，你想再经历一次？`
14. `这一分钟也在悄悄发生`
15. `你拍的那张图，以后会有用的`
16. `做一件小事也是度过时间的方式`
17. `慢一点，也没关系`
18. `一些时间适合什么都不做`
19. `今天也算`
20. `被你留下来的一小段，不会再走了`

**撰写原则（重要）：** 所有文案采用**观察式**语气而非"你应当..."。不做道德评判、不设定情绪、不用 guilt 触发行动。"温柔·成熟·不煽情"。

### Widget 轮换算法（给工程师）

```
每日凌晨 4:00 刷新一次：
1. 从用户已有的时间账户里，挑一个"本周相关性最高"的时间账户（条件：最近存入 or 最近一次见面天数）
2. 从该时间账户的文案池中随机抽一条，排除最近 7 天出现过的
3. 30% 概率用"今日觉知池"覆盖（保持新鲜）
4. 若用户已存入时刻 > 5 条，偶尔引用用户自己存的标题（"你上次存入的瞬间：xxx"）
```

### Widget 文案长度规则

| Widget 尺寸 | 最长字数 |
|------------|---------|
| Lock 胶囊（Circular） | 仅数字 + 单字单位 |
| Lock 矩形 | 20 字以内 |
| 桌面小 | 30 字以内 |
| 桌面中 | 40 字以内 |
| 桌面大 | 60 字以内 |
| StandBy | 25 字以内 |

## 四、通知文案池（V1.1 双池结构）

**核心规则（PRD §15 对齐）：**
- V1 每天最多一次（默认 9:00）
- **默认池 = neutral**（日常觉知型），是 opt-in 时默认就开的
- **关系池 = opt-in**，用户在"设置 → 通知 → 关系型提醒"里主动开启才会出现
- 开启关系池后，每天 1 次的配额不叠加，按一定概率（建议 30%）用关系池替换当日 neutral

### 4.1 默认池：日常觉知型（V1 启用，N = 12）

1. 你今天又看到了一次太阳升起。
2. 早。过去 24 小时有什么想存下来的吗？
3. 现在这一刻，是你以后的"过去"。
4. 窗外天气怎么样？记得看一眼。
5. 这是你的第 12,483 个早晨。
6. 喝口水，慢一点开始今天。
7. 今天的光，刚好落在今天。 ← 原"以后不会再有了"（tone 已降档）
8. 今天也算。
9. 最近的哪一天，你想再经历一次？
10. 不必记得所有，留下一两件就好。
11. 一些时间适合什么都不做。
12. 慢一点，也没关系。

### 4.2 关系池：opt-in 型（V1.1 启用，V1 预留但关闭）

**V1 设置里不出现这个开关**（完全不做，也就不会出现给用户 guilt）；V1.1 时才加。

设置开关文案：
> **关系型提醒**（默认关闭）
>
> 开启后，偶尔会收到跟家人、伴侣相关的温柔提醒（约每周 1-2 次）。
> 我们不会用"你已经 X 天没陪他们了"这种句子。
>
> [toggle] 开启

文案池（V1.1 启用，N = 8）：
1. 距离上次见爸妈，已经过了一些天。 ← 原"你上次见面是 32 天前"（避开精确天数 + guilt）
2. 今天是爸爸 63 岁生日。如果方便，可以打个招呼。
3. 1 年前的今天，你存入了"杭州西湖 48 小时"。
4. 下次见面的时候，也许会有想留下的一小段。
5. 你上次存入的瞬间是 {dim.lastMomentTitle}，{N} 天前。
6. 伴侣的生日在下个月。
7. 今天正好是 {某位家人} 的生日前 7 天。
8. 一年里还没到你和爸妈的下一次见面，没关系。

### 4.3 月度回顾型（V1.5+，每月最后一天）

- 2026 年 3 月，你存入了 7 小时 · 4 个瞬间。
- 3 月过完了，你在这里留下了 4 个片段。

## 五、文案审核 Checklist

任何上线文案提交前，自问：

- [ ] 读出声，像不像一位低声的朋友？
- [ ] 有没有用"！"？（能不用就不用）
- [ ] 有没有用 emoji？（V1 只在时间账户 icon 用 ♡ ◐ 这种符号，正文一律无 emoji）
- [ ] 有没有用"哇"、"棒"、"加油"之类的鼓舞词？
- [ ] 读完是"被理解"还是"被教育"？
- [ ] 数字前面有"约"字吗？（除非是精确的瞬间计数）
- [ ] 有没有默认用户情绪应该是什么？（不要预设用户开心/难过）

## 六、Formatter Matrix（与 PRD §21 完全一致）

> 所有 UI 文本展示必须调用 Formatter 接口，**不允许在 View 层手写单位拼接**。

### 语义单位速查

| Semantic | 示例 | 使用场景 |
|----------|------|---------|
| `hoursCompact(h)` | `552h` / `7,488h` / `18.2Kh` | 卡片主数字、Widget |
| `hoursReadable(h)` | `128 小时` / `48 小时` | 主页顶部总账户、时刻详情 chip |
| `hoursWithMinutes(s)` | `2h 30m` / `30m` | 新增时刻表单时长滑块、参数编辑 |
| `occurrenceCount(n, noun)` | `约 92 次见面` | parents 卡副文案（消耗层） |
| `weeklyHours(h)` | `每周约 30 小时` / `每周约 5 小时` / `每周约 40 小时` | kids / sport / create 卡副文案 |
| `dailyHoursWith(h, action)` | `每天约 4 小时共处` | partner 卡副文案 |
| `percentOfAwake(pct)` | `占清醒时间约 56%` | free 卡副文案 |
| `lifespanSubtitle(years, hoursK)` | `45 年 · 473 Kh` | 顶部时间余额卡副文案 |
| `momentsCount(n)` | `12 个瞬间` / `1 个瞬间` | **存储层所有场景的唯一量词** |
| `relativeTime(date)` | `3 天前` / `1 年前的今天` / `发生在 3 小时前` | 时刻详情、时间线 |
| `absoluteDate(date)` | `2026-03-22` | 时刻详情二级 |

### 存储层严格禁用词

| ❌ 禁用 | ✅ 替换 |
|------|------|
| 已存入 12 次 | 已存入 12 个瞬间 |
| 存了 5 次 | 存了 5 个瞬间 |
| 你的打卡 | 你的瞬间 |
| 记录（名词）| 瞬间 / 时刻 |

"次" 只能出现在消耗层副文案里（"约 92 次见面"）描述预期事件次数。

## 七、多语言（V1.5 计划）

英文版原则：**直译会死，重写才对。**

例如：
- 中文："你和爸妈还能一起度过约 552 小时"
- ❌ 直译：`You can spend about 552 more hours with your parents`（冷）
- ✅ 重写：`You still have about 552 hours with Mom and Dad.` ("Mom and Dad" 比 "parents" 有温度)

英文 Tone 参考：
- Headspace 的正文
- Calm 的通知文案
- Day One 的空态

不要参考：Duolingo（太欢乐）、LinkedIn（太商务）、Notion（太中性）。
