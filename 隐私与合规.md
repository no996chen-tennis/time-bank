# 时间银行 — 隐私与合规

> Version: 1.0 | Date: 2026-04-19 | 配套 PRD V1.2
>
> 本文是上架 App Store 前的 **自我合规体检** 和 **用户信任资产** 的建设说明。
>
> 核心原则：**隐私不是合规负担，是产品的第一级护城河。**
> 竞品 Life Clock 类产品几乎都在卖焦虑、求订阅，时间银行要反着做 —— 把"不收集数据"变成营销主张。

---

## 一、核心隐私承诺（Top-Level Promises）

以下 5 句话是对用户的**硬承诺**，必须出现在：
1. App Store 商店描述
2. 引导流程 Step 3 底部
3. 设置页「你的数据去哪了」入口
4. 隐私政策页面正文

```
 1. 所有照片、视频、文字都存在你自己的 iPhone 上。
 2. 没有账号。打开就能用。
 3. 没有任何第三方 SDK 看你的数据。
 4. 开启 iCloud 同步（V1.5）后，数据只存在你自己的 iCloud 私有库，
    Apple 端到端加密，连开发者都看不到。
 5. 你可以随时一键打包带走所有数据。
```

## 二、App Store Review Guideline 对照自查

针对 iOS App Review 最容易被卡的条款，每一条都要能清晰回答。

### Guideline 5.1 — Privacy（最严格）

| 条款 | 要求 | 时间银行如何满足 |
|------|------|----------------|
| 5.1.1(i) | 明确告知为何收集数据 | V1 不收集任何数据，`PrivacyInfo.xcprivacy` 全勾 No |
| 5.1.1(ii) | 最小化原则 | 只有生日、家人年龄存本地，从不上传 |
| 5.1.1(iii) | 必须有可访问的 Privacy Policy URL | 上线前在 GitHub Pages 或自有域名搭一个 |
| 5.1.1(iv) | 未登录态功能必须可用 | V1 本来就没有账号 |
| 5.1.1(v) | 不得对未注册用户施压 | 从头到尾没有注册流程 |
| 5.1.2 | 第三方数据共享必须标明 | **没有第三方 SDK**（连 Crashlytics 都不上） |
| 5.1.3 | 健康数据相关额外条款 | V1 不碰 HealthKit |

### Guideline 2.1 — App Completeness

- [ ] 所有功能在审核账号上可用（无需特殊解锁）
- [ ] 不能出现 Beta 字样或"Coming Soon"
- [ ] 崩溃率 < 0.5%

### Guideline 4.0 — Design

- [ ] 真机能跑，不依赖 Mac 模拟器 bug
- [ ] 符合 Human Interface Guidelines
- [ ] 没有用 Web view 做核心功能

### Guideline 3.1.1 — In-App Purchase

V1 没有 IAP，跳过。V1.5+ 若做 Pro 订阅，必须走 Apple IAP，不允许外部支付链接。

## 三、Privacy Manifest（`PrivacyInfo.xcprivacy`）

iOS 17+ 必须包含 Privacy Manifest 文件。完整配置如下：

```xml
<?xml version="1.0" encoding="UTF-8"?>
<!DOCTYPE plist PUBLIC "-//Apple//DTD PLIST 1.0//EN"
  "http://www.apple.com/DTDs/PropertyList-1.0.dtd">
<plist version="1.0">
<dict>
  <!-- V1 承诺不收集任何数据 -->
  <key>NSPrivacyTracking</key>
  <false/>

  <key>NSPrivacyTrackingDomains</key>
  <array/>

  <key>NSPrivacyCollectedDataTypes</key>
  <array/>

  <!-- Required Reason API 声明：我们为什么用这些 API -->
  <key>NSPrivacyAccessedAPITypes</key>
  <array>
    <!-- 用 FileTimestamp 来展示"X 天前" -->
    <dict>
      <key>NSPrivacyAccessedAPIType</key>
      <string>NSPrivacyAccessedAPICategoryFileTimestamp</string>
      <key>NSPrivacyAccessedAPITypeReasons</key>
      <array>
        <string>C617.1</string> <!-- Display file timestamps to the user -->
      </array>
    </dict>
    <!-- 用 UserDefaults 存用户偏好 -->
    <dict>
      <key>NSPrivacyAccessedAPIType</key>
      <string>NSPrivacyAccessedAPICategoryUserDefaults</string>
      <key>NSPrivacyAccessedAPITypeReasons</key>
      <array>
        <string>CA92.1</string>
      </array>
    </dict>
    <!-- 用 DiskSpace 显示"你占用了 X GB" -->
    <dict>
      <key>NSPrivacyAccessedAPIType</key>
      <string>NSPrivacyAccessedAPICategoryDiskSpace</string>
      <key>NSPrivacyAccessedAPITypeReasons</key>
      <array>
        <string>85F4.1</string> <!-- Display to the user disk space info -->
      </array>
    </dict>
  </array>
</dict>
</plist>
```

## 四、Info.plist 权限描述文案

**核心原则：** 每一条权限描述都要真实、具体、短。Apple 审核员读到觉得"合理"就过。

```
NSCameraUsageDescription
值：用相机拍下当前的瞬间，存进你的时间银行。照片只会保存在你的 iPhone 上。

NSPhotoLibraryAddUsageDescription
值：不需要！我们不写入你的相册。（如不用请不要加此 key）

NSPhotoLibraryUsageDescription
值：不需要！我们用系统 PhotosPicker 选图，无需读取相册权限。（不用不加 key）

NSMicrophoneUsageDescription
值：视频录制需要声音，这样你能听见当时的对话。声音只会和视频一起存在你的 iPhone 上。

NSUserNotificationsUsageDescription
（通常不需要）
```

**严格不要加的权限：**
- ❌ NSPhotoLibraryUsageDescription（读取相册）—— 用 PhotosPicker 替代
- ❌ NSLocationWhenInUseUsageDescription —— V1 不用定位
- ❌ NSContactsUsageDescription —— V1 不读通讯录
- ❌ NSFaceIDUsageDescription —— V1 无需人脸识别
- ❌ NSHealthShareUsageDescription —— V1 不对接 HealthKit

**加这些权限会导致：** 用户首次弹窗时情绪抗拒 + 商店描述被质疑。

## 五、隐私政策模版（中文版）

放在独立 URL（建议 `https://timebank.adamchen.com/privacy`，或 GitHub Pages）。

---

**隐私政策**

_生效日期：2026-XX-XX · 版本：1.0_

### 一、我们是谁

时间银行是一款由独立开发者 Adamchen（下文简称"我们"）开发的 iOS 应用。

### 二、我们收集什么

**V1 版本：我们不收集任何数据。**

具体来说：
- 你在 App 内输入的生日、家人信息、存入的瞬间（包括文字、照片、视频），全部存在你自己 iPhone 的 App 沙盒里。我们无法访问。
- 我们不使用任何第三方统计 SDK。我们没有"后台"能看到你的使用数据。
- App 不联网（V1）。

### 三、你的数据在哪里

- **所有内容** 都存在你 iPhone 的 App 沙盒目录 `Documents/TimeBank/` 下。
- **媒体文件**（照片、视频）是原始文件，不经过任何压缩、重编码或服务器中转。
- 删除 App 会删除以上全部内容。我们**无法**恢复。

### 四、iCloud 同步（V1.5 开始）

如果你主动开启 iCloud 同步：
- 数据会同步到 **你自己的 iCloud 账号** 的 Private Database。
- 这是 Apple 原生的 CloudKit 服务，端到端加密，**连我们开发者也看不到**。
- 占用 **你自己的 iCloud 存储配额**。
- 关闭 iCloud 同步后，数据仍保留在你的设备上。

### 五、相机与麦克风权限

- 只在你主动点"拍摄"按钮时触发。
- 拍摄的内容直接存入 App 沙盒，不上传任何服务器。
- 你可以随时在 iOS 设置中关闭这些权限。

### 六、通知

- 你可以选择开启"每日温柔提醒"。
- 通知内容由 App 本地生成，不经过我们的服务器。
- 随时可以在系统设置关闭。

### 七、儿童隐私

时间银行不面向 13 岁以下儿童。如果你是未成年人的监护人并发现孩子在使用本 App，请联系我们，我们会协助你删除相关数据。

### 八、数据导出与删除

- **导出**：设置页 → "导出全部数据"，生成一个 ZIP，含你所有内容。
- **删除**：设置页 → "清空时间银行"，或直接删除 App。

### 九、本政策的更新

我们会在版本更新时更新本政策。重要变更会通过 App 内通知告知。

### 十、联系我们

如果你对隐私有任何问题，发邮件：privacy@adamchen.com

## 七、中国大陆特别合规（上架中国区必看）

### 《个人信息保护法》适配

- [ ] 隐私政策在"首次启动"必须展示给用户并获得**明示同意**
- [ ] 隐私政策放在中国大陆可访问的域名（避开 Vercel/Netlify 这种有时被墙的服务）
- [ ] 用户可以"一键撤回同意并退出 App"

### App 备案

2024 年起，中国区上架的 App 需要 **工信部 ICP 备案号**。
- 如 Adam 个人没备案域名，需要新备案（2-3 周）
- 或：注册 **Apple Developer 企业账号**（门槛高、不推荐个人开发者）
- 或：**先不上中国区**，只上海外 App Store（美区为主）

### 关于"算法推荐"

V1 没有算法推荐（Widget 轮换不构成推荐算法），可跳过。

### 关于"数据出境"

V1 数据不出境（本地存储）。V1.5 iCloud 同步时需加一句：
> "iCloud 服务由 Apple 在中国运营，数据存储在中国境内（详见 Apple 官方说明）。"

### 审核层

- [ ] 准备好回答 App Review 可能的 5 个典型问题（见下）

### 典型审核问题预演

**Q1: "Your app stores user personal information. Please clarify how you handle this."**
> A: All user data (birthday, family info, moments) is stored exclusively in the app's sandboxed Documents directory on the user's device. We do not transmit any data to external servers. The app functions entirely offline in V1.

**Q2: "Does your app use any third-party SDKs that collect user data?"**
> A: No. The app has zero third-party SDKs. No analytics, no crash reporting, no advertising frameworks.

**Q3: "You request Camera permission — clarify usage."**
> A: Camera is only accessed when user explicitly taps the "Take photo/video" button within the Moment creation flow. Captured media is saved directly to the app's sandbox, not to Photos library, not uploaded.

**Q4: "You use PhotosPicker — do you need Photos library permission?"**
> A: No. We use the system PhotosPicker API (iOS 16+) which requires no runtime permission. Users select specific images/videos through the system UI.

**Q5: "Does the app function without signing in?"**
> A: The app has no sign-in flow at all. All users use it anonymously.

## 九、用户信任资产（把隐私做成营销）

在以下触点**反复强调**隐私承诺：

### 商店截图第 2 张
背景：米白色 + 大字：
> **你的数据，全部在你自己的 iPhone 上。**
> 不上传、不联网、不需要账号。

### 引导 Step 1 底部
`这些只会存在你的手机里。` （12pt，柔和）

### 设置页「你的数据去哪了」
一个入口，点进去是上面的隐私政策。但标题故意不叫"隐私政策"，叫 **"你的数据去哪了"**。

### 商店描述结尾
> **一句承诺：**
> 你的时间，你的数据。时间银行只是一个容器。
> 不收集、不分析、不上传。
> 你关 App 的那一刻，它什么都不会带走。

### 小红书/独立开发者圈种子文案
> 我做了一个 App，没有任何后端、没有任何 SDK、没有账号系统。
> 代码量小到一个人能维护。
> 因为我相信 —— 一个记录"和爸妈相处的时间"的工具，绝不应该把这些发到任何服务器。

这段文字会在开发者社区产生强共鸣，是冷启动的关键素材。

## 十、事故预案

### 如果不慎误加了数据收集代码

1. 24 小时内紧急发版本修复
2. App 内公告 + 邮件通知用户
3. 在隐私政策更新说明并坦诚告知

### 如果 App Review 要求加数据收集

不要为了过审加任何非必要的 SDK。
- 如果被要求，先沟通
- 如果确实被拒，考虑更换方案（例如某个 SDK 换原生实现）
- 绝不默默加上

### 如果用户发现 Bug 导致数据丢失

1. 立即公告承认
2. 发补丁
3. 已有版本的所有用户自动触发"一键导出备份"流程

**把透明度做到极致是唯一的出路。** 一款关于"时间"的 App，如果丢了用户的数据，所有信任瞬间归零。

## 十一、最后一句

V1 产品上线时，"什么都不收集"不是 bug，是 feature。
在一个所有 App 都在抢着收数据的时代，这是最强的差异化。
