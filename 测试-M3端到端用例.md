## 0. 测试前置

### 0.1 环境

- iOS 模拟器：iPhone 17 Pro · iOS 17+
- 设备：Light Mode（V1 无 Dark Mode）
- 网络：可断网（V1 全离线）
- 时区：默认 `Asia/Shanghai`
- 时间：以模拟器系统时间为准（不要手动调，避免 relative time 跑偏）

### 0.2 重置 App 状态

每次切换 Persona 或 Script 前必须重置 App，方法二选一：

- **方法 A（推荐）**：删除模拟器上的 TimeBank App + 重新 build install → 触发 onboarding
- **方法 B**：在 Xcode `xcrun simctl uninstall booted com.adamchen.timebank`（替换 bundle id）→ rebuild

> **不要**仅靠"清理数据"按钮（V1 暂未实现 UC-7.3）。

### 0.3 报告格式（Codex 输出要求）

每条 TC 跑完后输出：

```
### TC-XXX: <用例名>
- 状态: ✅ PASS / ❌ FAIL / ⚠️ PARTIAL
- 步骤完成: N/M
- 截图: [截图描述或路径]
- 偏离/异常: <如有，描述用户实际看到的 vs 预期>
- 严重度: P0 阻塞上架 / P1 严重 / P2 一般 / P3 可接受
```

最后汇总成 `测试报告-MM-DD-HH.md` 放在仓库根，命名包含日期和小时。

## 3. P1 核心路径（50+ 条）

### 3.1 Onboarding 分支

#### TC-O01 · 单身路径（仅勾 solo）

按 Persona B 跑 → 完成后主页只展示运动/创造/自由 3 张时间账户卡（关系类的 parents/kids/partner 应该 hidden）。

#### TC-O02 · 部分关系（只勾父母+伴侣）

跳过孩子勾选 → 主页应该没有"陪孩子"卡片。

#### TC-O03 · Step 3 频率单位切换

在 Step 3 父母频率字段：
1. 默认「每年」
2. 输入 12 → 切「每月」→ 应该变 1（12/12 = 1）
3. 切「每周」→ 应该变 0 但 max(1) 保护变 1
4. 切回「每年」→ 显示 52（1×52）

> 已知 round-trip 漂移：12 → 1 → 1 → 52，不再是原来的 12。这是预期行为，不算 bug。

#### TC-O04 · 出生年输入边界

YearInputField 行为：
- 输入 1900 → 失焦后被 clamp 到 1920（lower bound）
- 输入 2099 → 失焦后被 clamp 到当前年（如 2026）
- 清空字段 → 不应跳到 upper bound（B1-γ polish 修过）/ 失焦回到上一合法年份
- 输入字母 → 应被 filter 掉

#### TC-O05 · 关系全空（V1 不允许）

理论上 onboarding step 2 至少要勾一个 → 试着全部不勾 → 「下一步」按钮应该 disabled，或勾选 solo 才能继续。

> 实际行为：solo 默认勾选，所以"全空"做不到。验证 solo 默认勾选即可。

### 3.2 主页

#### TC-H01 · 主页 0 瞬间空态副文案

新装 App 完成 onboarding 后，TotalAccount 双层卡的「已存入」副文案应该是 §2.3 的空态文案：「还没有被存入的时刻。下次有值得留下的片段，来这里看看。」

#### TC-H02 · 卡片 ≥24h「≈ N 天」辅助行

存入一条时长 100h 的瞬间 → 主页该卡片消耗层数字下方多一行「≈ 4 天」。

#### TC-H03 · 主页 6 卡片排序与勾选

切 Persona D（仅母亲）→ 主页应该看到「陪父母」（仅母亲）+「陪孩子」+ 运动/创造/自由 5 张卡（无伴侣）。

### 3.3 详情页

## 6. 已知 limitation（不是 bug · Codex 不要误报）

| 行为 | 说明 |
|---|---|
| 「你的数据去哪了」/「关于时间银行」点击无响应 | A2 stub · M4 才接入完整页面 |
| 「FAB」点击在标记父母 memorial 时仍能打开 MomentEditor | FAB 是全局入口，dimension picker 排除 memorial 即可 |
| 频率单位切换 round-trip 数字漂移（12/年→1/月→52/年）| 整数除法精度损失，不算 bug |
| 单条删除快速连续 3 次只能撤销最新 | UndoToastController 设计为单 slot，前面的 Undo 机会丢失（5s Timer 各自跑，不影响数据正确性）|
| 编辑器场景缓存可能内存峰值高 | B1-γ N-1 已优化为 256×256 降采样，但极端情况（一次编辑 9 张大图）仍可能 ~200MB peak |
| 账户 Tab 点击环形图段不进单段详情页 | V1 不实现，仅展示分布 |
| MomentDetail 标记纪念账户中的 + 按钮 | 详情页 + 已 disabled，但 timeline 入口（如长按 row 进批量再单选）某些路径可能绕过——若发现请记录路径 |
| 设置页缓存清理 / 清空数据 | M4 才接入 |

## 8. Codex 执行规范

1. **顺序**：P0 烟雾全过再继续 P1；P1 失败别 block P2，但要标 P1 失败；P3 视觉用截图，不阻塞 P0/P1
2. **每个 Persona 独立 cold install**：不要在一个 install 里串多个 Persona 数据
3. **截图节点**：每个 P0 用例至少 3 张关键截图（前 / 操作中 / 后），P1/P2 至少 1 张关键截图
4. **失败时立刻截图**：不要继续操作覆盖证据
5. **时序敏感**：Undo 5s 窗口、慢保存 10s、自动消失 1.6s 这些用 simctl 等待或 wait 精确控制
6. **报告语言**：中文，跟项目其他文档一致
7. **报告 commit**：跑完后 `git add 测试报告-YYYY-MM-DD-HH.md && git commit -m "test: M3 端到端测试报告 YYYY-MM-DD"` 推到 m3-app-loop
