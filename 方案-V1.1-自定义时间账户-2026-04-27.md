### C. 计算方式 spec

PRD §5 L229："计算方式（每周 X 小时 或 每年 X 次 等）"。本方案具体化为 3 种 preset：

| Preset | params struct | 适用场景 | UI |
|---|---|---|---|
| **每周 N 小时** | `weeklyHours: Double` | 阅读 / 旅行 / 冥想 / 学习 / 通勤 | slider 0-80 step 0.5 |
| **每天 N 小时** | `dailyHours: Double` | 类似伴侣的 daily commitment | slider 0-16 step 0.5 |
| **每年 N 次 · 每次 M 小时** | `annualOccurrences: Int, hoursPerOccurrence: Double` | 类似父母的"见面型"账户：朋友聚会、年度旅行 | 2 个 picker（次数 1-365 / 时长 0.5-72） |

3 个 preset 用 enum + associated value 持久化到 `Dimension.params: Data`。

新增 `CustomDimensionParams` schema（**会改 PRD §7.6**）：

```swift
enum CustomFormula: String, Codable, Sendable {
    case weeklyHours
    case dailyHours
    case occurrenceBased
}

struct CustomDimensionParams: Codable, Sendable, Equatable {
    var formula: CustomFormula
    var weeklyHours: Double          // formula == .weeklyHours 时有意义
    var dailyHours: Double           // .dailyHours
    var annualOccurrences: Int       // .occurrenceBased
    var hoursPerOccurrence: Double   // .occurrenceBased

    init(
        formula: CustomFormula = .weeklyHours,
        weeklyHours: Double = 5,
        dailyHours: Double = 1,
        annualOccurrences: Int = 12,
        hoursPerOccurrence: Double = 2
    ) { /* ... */ }
}
```

`DimensionCompute.consumeHours` 加一个 `case dimension.kind == .custom` 分支：

```swift
let p = dimension.decodeParams(CustomDimensionParams.self, default: CustomDimensionParams())
let remainingYears = Lifespan.remainingYears(profile: profile, now: now)
switch p.formula {
case .weeklyHours:
    return remainingYears * weeksPerYear * max(0, p.weeklyHours)
case .dailyHours:
    return remainingYears * daysPerYear * max(0, p.dailyHours)
case .occurrenceBased:
    return remainingYears * Double(max(0, p.annualOccurrences)) * max(0, p.hoursPerOccurrence)
}
```

`DimensionCompute.subtitleData` 同步加 case 返回对应的 `.weeklyHours` / `.dailyHoursWith` / `.occurrence`。

### F. 主页编辑模式（新增 · Adam 2026-04-27 决策 · iOS jiggle 风）

**两个触发入口**：
1. **显式**：主页"时间账户 · N 个"标题右侧「编辑」按钮 → 切到编辑模式
2. **隐式（iOS 主屏标准）**：长按任一卡片 ≥0.5s → 切到编辑模式
   - 用 `.onLongPressGesture(minimumDuration: 0.5)` 触发
   - 触发瞬间用 `UIImpactFeedbackGenerator(style: .medium).impactOccurred()` 给一个 haptic 触感
   - 触发后**保持原卡片视觉**（不是把它放大），直接整组卡片同时进入抖动

**编辑模式 UI 变化**：
- 顶部双层卡（lifespan + 已存入）半透明 dimmed 0.4，禁止交互
- 副标题"时间账户 · N 个"改为"长按拖动 · 调整顺序"
- 右上 `[编辑] [+]` 两个按钮替换为单一胶囊按钮 `[完成]`（主色填充）
- 卡片**保持原有结构**（双数字 / icon / 名称都还在），只是叠加抖动 + 自定义账户左上角浮一个 − 按钮
- **抖动动效**（iOS 主屏 jiggle 标准）：
  - 旋转 ±1° 到 ±1.5° 之间小幅来回
  - 周期 0.28-0.32s（不要太快显得焦躁）
  - 每张卡片用 random phase delay 0.0-0.15s，不要齐刷刷一起抖
  - 抖动用 `.rotationEffect(.degrees(angle)) + .animation(.easeInOut.repeatForever(autoreverses: true))`
- **删除按钮**：自定义账户左上角浮一个圆形「−」按钮（黑底白边白十字，跟 iOS 主屏 app 删除按钮一致）
- 卡片下方虚线"+ 添加自定义时间账户"按钮在编辑模式下隐藏

**退出编辑模式**：
- 点「完成」按钮 → 写入新 sortIndex 到 SwiftData → 抖动平滑停下 → 切回普通模式
- 点抖动卡片以外的空白区域 → 同上（标准 iOS 行为）
- 删除自定义账户走 `DimensionDemotionStore.demote(dimensionID:)`，弹 confirm dialog 后转 other

**实施伪代码（SwiftUI）**：
```swift
@State private var editMode: HomeDimensionEditMode = .inactive

ForEach(visibleDimensions, id: \.id) { dimension in
    DimensionCardView(dimension: dimension, ...)
        .modifier(JiggleEffect(isActive: editMode == .editing,
                               phaseDelay: Double(index) * 0.04))
        .overlay(alignment: .topLeading) {
            if editMode == .editing && dimension.kind == .custom {
                DeleteBadge { showDeleteConfirm = dimension }
                    .offset(x: -8, y: -8)
            }
        }
        .onLongPressGesture(minimumDuration: 0.5) {
            UIImpactFeedbackGenerator(style: .medium).impactOccurred()
            withAnimation(.spring(response: 0.4, dampingFraction: 0.7)) {
                editMode = .editing
            }
        }
        .allowsHitTesting(editMode == .inactive)  // 编辑模式禁用进详情
}

struct JiggleEffect: ViewModifier {
    let isActive: Bool
    let phaseDelay: Double  // 每张卡片错开
    @State private var angle: Double = 0

    func body(content: Content) -> some View {
        content
            .rotationEffect(.degrees(isActive ? angle : 0))
            .onAppear { startWobble() }
            .onChange(of: isActive) { _, _ in startWobble() }
    }

    private func startWobble() {
        guard isActive else { angle = 0; return }
        let target = Double.random(in: 1.0...1.5) * (Bool.random() ? 1 : -1)
        DispatchQueue.main.asyncAfter(deadline: .now() + phaseDelay) {
            withAnimation(.easeInOut(duration: 0.30).repeatForever(autoreverses: true)) {
                angle = -target
            }
        }
    }
}
```

**拖动排序实施**：
- iOS 17+ 用 `.draggable(dimension.id)` + `.dropDestination(for: String.self)`
- 或最简方案改 `List` + `.onMove` + `.editMode = $editMode`
  - **推荐 List 方案**：原生支持，最少坑
  - 但当前 HomeView 用 ScrollView + VStack，改 List 可能破坏首屏 layout
  - Codex 评估：先试 List + `.listStyle(.plain) + .listRowBackground + .listRowSeparator(.hidden)`，如果不能 1:1 复刻当前视觉，再回到 ScrollView 自己实现 `.draggable`
- 排序拖动结束后 batch update：`for (index, dim) in array.enumerated() { dim.sortIndex = index }` + `try modelContext.save()`
- 内置账户也参与排序（用户可以把"运动"拖到"陪父母"上面）；只是不能删
- lifespan（systemTop）不在排序范围（永远在最顶部，编辑模式下半透明）

**约束**：
- 编辑模式下底部 + 按钮和右上 + 按钮都隐藏（避免新建后排序混乱）
- 编辑模式下卡片不响应 navigation tap（只响应拖动 / 删除）
- 编辑模式持续时间内禁用 navigation pop（用户必须显式「完成」或点空白才退出）
- lifespan / 已存入双层卡 dimmed 0.4，不抖动

**风险点（Codex 审）**：
- List + custom row styling 比 ScrollView + VStack 难做 100% 一致的视觉。可能要用 `.listStyle(.plain) + .listRowBackground + .listRowSeparator(.hidden)` 复制现有卡片间距
- 如果改 List 影响 PRD §22 layout 硬约束（首屏 ≥4 张卡），保持 ScrollView，自己实现 `.draggable` 拖放
- jiggle 抖动是连续动画，可能在低性能机器上耗电；进入编辑模式自动 6s 无操作可考虑暂停抖动（V1.1 不做，V1.2 polish）
- `.onLongPressGesture` 和 `NavigationLink` 共存可能冲突，需测试：长按触发 jiggle 而非 navigate
- `.allowsHitTesting(false)` 阻断 navigation 时，长按手势仍要能识别（通常需要 simultaneousGesture）

## 主要风险点 + Codex 审核重点

### R1. 颜色 palette refactor 涉及 6 个文件

`DimensionPalette.color(for: id)` 改成 `color(forColorKey:)` 是 breaking change，所有调用点必须同步改。Codex 必须 grep 全项目，**漏一处颜色就错位**。Suggestion: 旧签名 `color(for: id)` 暂时保留作为 fallback（id 命中保留路径），用 `@available(*, deprecated)` 标记，下个版本删。

### R2. 自定义账户 default 文案分支

`DimensionDetailCopy` 中 `headerSubtitleLines` / `insight` / `calculationSummary` 现在 default case 返回 `[]` 或 `""`，自定义账户走的就是 default。这会导致详情页副文案空白。需要给 default 分支写**通用文案**：
- `headerSubtitleLines` default：用 dimension.name + computed 数字，如 "约还能 \(name) · 共 N 周" + Formatter.weeklyHours/dailyHoursWith/occurrenceCount 对应分支
- `insight` default：返回 nil（不强制每个自定义账户有 insight）
- `calculationSummary` default：用 formula 推导，如 "每周 \(p.weeklyHours) 小时"

### R3. 主页 layout 测试

主页 `visibleAccountDimensions` 已经包含 `.custom`，但 PRD §22.3 有 layout 硬约束（"iPhone 16 Pro 852pt 屏首屏可见双层卡 + ≥4 张卡"）。如果用户加了 10 个自定义账户，主页会变得很长，但首屏 ≥4 张卡的约束仍然满足。**不需要改 layout**，只是 ScrollView 内容变多。Codex 验证一下加 5 个自定义账户后主页还能正常滚动。

### R4. 与 visual rebrand 的耦合

下一步计划 visual rebrand。如果 rebrand 重新设计了配色系统（比如 V3 设计探索的 Apartamento / Swiss / Zen 风格各自有不同色板），现在加的 10 色可能要重做。**Codex 实施时要保证颜色定义集中在 `DesignTokens.swift`**，不要散到组件里，方便 rebrand 一次替换。

### R5. PRD §7.6 schema 改动

加 `CustomDimensionParams` 是新增字段，**不破坏既有 SwiftData migration**。但 Codex 要在 PR 里同步更新 PRD §7.6 文档（新增字段必须先改 schema 再改代码 — PRD 原话 §7.6 line 569）。

### R6. 现有的"自定义时间账户"测试断言

`DataLayerTests.swift` L541-588 已有 customDimension 用例，但用的是 `EmptyDimensionParams()`。新方案下应该用 `CustomDimensionParams`。Codex 要更新这个测试用例。

### G. 创建 sheet 预览卡位置（**Adam 2026-04-27 决策**）

**最终方案**：sheet **顶部 sticky** 显示预览卡（绿色），名称/图标/颜色/公式 fields 在下方滚动。

**理由**：用户调 slider 时能立刻看到"每周 5 → 6 小时"和"1.4 万小时 → 1.7 万小时"实时跳动，反馈强。

**实施细节**：
- `DimensionEditorView` 用 `VStack` 而非 `ScrollView`：顶部固定 preview card，下方 ScrollView 装 fields
- preview card 始终读 draft 当前 state 实时计算
- 首次渲染时（无任何 fields 输入）显示占位 "0 小时" + "保存后看效果"
