# 时间银行 — 从零到上线完整指南

> 面向完全没做过 iOS 开发的人。预计总周期：10-12 周。

---

## 准备清单（开发前必须搞定）

### 硬件

| 项目 | 说明 | 你的情况 |
|------|------|----------|
| Mac 电脑 | 必须。M1 及以上芯片最佳，8GB+ 内存，50GB+ 空闲空间 | ☐ 已有 |
| iPhone | 用于真机测试。Widget 必须在真机上测试，模拟器不够用 | ☐ 已有 |
| USB-C / Lightning 数据线 | 连接 Mac 和 iPhone | ☐ 已有 |

### 软件 & 账号

| 项目 | 说明 | 耗时 | 费用 |
|------|------|------|------|
| **Xcode** | Mac App Store 免费下载，~15GB | 30 分钟安装 | 免费 |
| **Apple Developer Program** | developer.apple.com 注册 | **24-48h 审批** | **$99/年** |
| **App Store Connect** | 注册开发者后自动开通 | — | 免费 |
| **Figma** | 设计工具，figma.com 注册 | 10 分钟 | 免费版够用 |

### ⚠️ 必须提前做的事

**Apple Developer Program 申请要立刻做！** 这是整个流程的最大卡点。没有开发者账号，你可以在模拟器上开发和调试，但无法在真机上测试 Widget、无法上 TestFlight、无法提交 App Store。审批一般 24-48 小时，但偶尔会更久。

申请流程：
1. 打开 developer.apple.com → Enroll
2. 用你的 Apple ID 登录
3. 选择"个人"类型（不需要公司）
4. 填写真实姓名、地址、税务信息
5. 支付 $99
6. 等待审批邮件

## 完整开发时间线

### 第 0 周（现在立刻做）

**并行任务——同时进行：**

```
[任务 A] 申请 Apple Developer Program ——→ 等 24-48h 审批
[任务 B] 安装 Xcode ——→ 打开跑个 Hello World
[任务 C] Figma 做 3 个设计方向 mockup ——→ 选定方向
[任务 D] 确认 PRD 中的 Open Questions
```

### 第 1-2 周：学 SwiftUI + 定设计

**学习路线（每天 2-3 小时）：**

第 1-3 天：Swift 语言基础。变量、函数、结构体、枚举、可选值。不用学太深，够用就行。

第 4-7 天：SwiftUI 基础。VStack/HStack/ZStack 布局、Text/Button/Image 基础组件、@State 和 @Binding 数据绑定、List 和 NavigationStack 导航、Form 表单输入。

第 8-10 天：做一个练手项目。比如一个简单的计算器或待办清单 App，目的是把上面学的串起来。

第 11-14 天：学 SwiftData（数据存储）和 App Groups（Widget 数据共享）。

**推荐学习资源：**
- Apple 官方 SwiftUI Tutorials（developer.apple.com/tutorials/swiftui）
- Hacking with Swift — 100 Days of SwiftUI（hackingwithswift.com）
- YouTube 搜 "SwiftUI beginner 2025/2026"

**同时在做：** Figma 出 3 个方向设计稿 → 选定 1 个。

### 第 3-6 周：核心 App 开发

**按功能拆分：**

第 3 周 — 项目骨架 + 引导流程：
- 创建 Xcode 项目（选 App 模板 + SwiftUI）
- 搭建导航结构（TabView / NavigationStack）
- 做 3 步引导页（生日 → 家人信息 → 完成）
- 用 SwiftData 存储用户输入的数据

第 4 周 — 主页 + 时间计算引擎：
- 实现所有时间时间账户的计算逻辑（一个纯 Swift 函数文件）
- 主页卡片列表
- 卡片的视觉样式（颜色、图标、进度条）

第 5 周 — 详情页 + 参数编辑：
- 点击卡片进入详情
- 展示完整数据和洞察文案
- 参数编辑功能（修改后实时刷新数字）

第 6 周 — "今日此刻"模块 + 自定义追踪项：
- 日常时刻的横滑卡片
- "添加新时间账户"功能
- 全局设置页面

### 第 7-8 周：Widget 开发

这是你产品最核心的部分，值得花两周。

第 7 周 — 锁屏 Widget：
- 在 Xcode 中添加 Widget Extension target
- 配置 App Group（让 Widget 和 App 共享数据）
- 实现锁屏 Widget（小尺寸），展示一句话 + 数字
- 实现内容每日轮换逻辑

第 8 周 — 桌面 Widget + 打磨：
- 桌面 Widget（中号），展示多时间账户概览
- Widget 配置界面（用户选择偏好展示的时间账户）
- 真机测试所有 Widget 场景

### 第 9-10 周：打磨 + TestFlight

第 9 周 — 自测 + 修 bug：
- 真机上完整走一遍所有功能
- 检查各种边界情况（没有孩子怎么显示、父母年龄很大怎么处理等）
- 性能检查（App 启动速度、内存占用）
- 去掉所有 print() 调试代码

第 10 周 — TestFlight 内测：
- Xcode → Product → Archive → 上传到 App Store Connect
- 在 App Store Connect → TestFlight 中创建测试组
- 邀请 5-10 个朋友安装 TestFlight App 参与测试
- 收集反馈，修复关键问题

### 第 11 周：提交上线

**提交前检查清单：**

```
App Store Connect 元数据：
☐ App 名称（30 字以内）
☐ 副标题（30 字以内）
☐ 关键词（100 字以内，逗号分隔）
☐ 完整描述（4000 字以内）
☐ App 图标（1024×1024 PNG，无透明）
☐ 截图（至少 3 张，推荐 5 张，需要 6.7" 和 6.1" 两个尺寸）
☐ 隐私政策 URL（必须！用 Notion 公开页面就行）
☐ 支持 URL（可以用邮箱链接）
☐ 年龄分级（填问卷，5 分钟）
☐ 价格设为"免费"
```

**提交后：** Apple 审核一般 24-48 小时。通过后状态变为"Ready for Sale"，App 即上架。

## 关键技术决策

### 技术栈

| 选择 | 理由 |
|------|------|
| **SwiftUI** | Apple 主推，代码量少，天然支持 Widget，学习曲线最友好 |
| **SwiftData** | 比 Core Data 简单很多，iOS 17+ 原生支持，适合新项目 |
| **WidgetKit** | iOS 原生 Widget 框架，支持锁屏 + 桌面 + StandBy |
| **App Groups** | App 和 Widget 共享数据的标准方案 |
| **最低支持 iOS 17** | SwiftData 需要 iOS 17，且 2026 年 iOS 17+ 覆盖率已超 90% |

## 常见踩坑提醒

**开发阶段：**
- Widget 不能实时更新，系统控制刷新频率。不要试图做秒级倒计时。
- App Group 的配置比较容易出错。第一次配置时按 Apple 文档一步步来。
- SwiftUI 的 Preview 功能很方便，但偶尔会卡死。重启 Xcode 解决。
- 真机测试时如果签名报错，Xcode → Signing & Capabilities → 勾选 "Automatically manage signing"。

**提交阶段：**
- 隐私政策是必需的，即使你不收集任何数据。写一个简单的"我们不收集任何数据"的页面也行。
- App 截图不要只是简单截屏。加上文字说明会显著提高下载转化率。
- 如果被拒，不要慌。Apple 会告诉你具体原因，修改后重新提交即可，一般 24h 内二次审核。
- 第一次提交被拒是非常正常的，大约 30% 的 App 首次提交会被拒。

**Widget 相关：**
- 很多用户不知道怎么添加 Widget。考虑在 App 内加一个简单的引导教程。
- 锁屏 Widget 只支持很小的尺寸，内容要极简。
- Widget 里不能有交互按钮（iOS 17+ 才支持简单的 Button）。
